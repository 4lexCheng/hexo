## 2. 环境准备

### 2.1 安装 Node.js

```bash
# 检查版本
node -v
npm -v
```

### 2.2 安装 Git

```bash
git -v
```

### 2.3 安装 Hexo CLI

```bash
npm install -g hexo-cli
```

{% note warning %}
如果下载很慢或失败，可以临时切换 npm 镜像源：
{% endnote %}

```bash
npm config set registry https://registry.npmmirror.com
```

## 3. 创建 GitHub 仓库

创建用于页面托管的仓库：

- 仓库名必须是：`用户名.github.io`
- 可见性选择：`Public`
- 可选：初始化 README

## 4. 配置 SSH（推荐）

### 4.1 生成密钥

```bash
ssh-keygen -t rsa -C "你的邮箱"
```

### 4.2 配置到 GitHub

- 复制 `id_rsa.pub` 内容
- 打开 GitHub：`Settings -> SSH and GPG keys`
- 新增 SSH key 并保存

### 4.3 测试连通性

```bash
ssh -T git@github.com
```

首次连接输入 `yes`。

## 5. 本地初始化 Hexo

{% note info %}
项目目录名尽量使用英文、数字和下划线，避免中文或特殊字符引发路径问题。
{% endnote %}

```bash
# 1) 初始化
hexo init

# 2) 安装依赖
npm install

# 3) 新建文章
hexo new "我的文章"

# 4) 生成静态文件
hexo generate

# 5) 本地预览
hexo server
```

本地访问：`http://localhost:4000/`（停止服务：`Ctrl + C`）。

## 6. 部署到 GitHub Pages

### 6.1 配置部署信息

编辑根目录 `_config.yml`：

```yaml
deploy:
  type: git
  repository: https://github.com/4lexCheng/4lexCheng.github.io.git
  branch: main
```

### 6.2 执行部署

```bash
hexo clean
hexo generate
hexo deploy
```

## 7. 常见报错与处理

### 7.1 报错：`ERROR Deployer not found: git`

原因：没有安装 Git 部署插件。

处理：

```bash
npm install hexo-deployer-git --save
```

安装后重新执行：

```bash
hexo clean
hexo generate
hexo deploy
```

### 7.2 部署成功但页面未更新

排查顺序：

- 确认仓库名是否为 `用户名.github.io`
- 确认 `deploy.repository` 和分支配置正确
- 等待 1-3 分钟后强刷浏览器缓存（`Ctrl + F5`）

## 8. 上线前检查清单

- Node.js / npm / Git 版本正常
- `_config.yml` 中 `url` 已配置线上地址
- `_config.yml` 中 `deploy` 配置正确
- 已安装 `hexo-deployer-git`
- 本地 `hexo server` 访问正常后再部署
